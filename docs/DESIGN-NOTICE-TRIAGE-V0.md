## Approaches Considered

### Approach A: Manual upload + AI draft (~3 weeks, minimal viable)

- Antonio (or his client) uploads a notice scan in Command Room
- AI OCRs the notice, classifies the type, pulls relevant authority + procedure
- Drafts response in Markdown editor inside Command Room with citations
- Antonio reviews/edits/approves
- Antonio copies the final response and files it manually in IRS Solutions
- **No browser automation in this approach.** Pure knowledge layer + classifier + drafter.

**Pros:** Fast, low risk, no integration brittleness, ships in month 1 with confidence. Validates the value prop (does Antonio actually use the drafts?) before expensive integration.

**Cons:** Antonio still files manually in IRS Solutions. Not the "holy shit" demo. Lower workflow displacement.

### Approach B: Full IRS Solutions integration (~5 weeks, ideal)

- Antonio's IRS Solutions credentials in Infisical credential vault
- Playwright drives IRS Solutions UI
- IAN (IRS Advance Notice) alerts pulled into Docket pipeline automatically
- Notice arrives → AI auto-classifies → pulls transcripts → drafts response → surfaces in Morning Brief
- Antonio approves in Command Room → automation files response back into IRS Solutions

**Pros:** True end-to-end. The "holy shit" moment for prospect demos. Highest workflow displacement value.

**Cons:** Brittle browser automation (UI changes break flows). Higher failure blast radius. 5 weeks is tight for month 1 alone.

### Approach C: Email-forward triage (~2 weeks, lateral)

- Antonio forwards IRS notices to `notices@docket.app`
- AI classifies, pulls authority, drafts response
- Reply email contains draft + "approve and email back" link
- Antonio approves → AI emails finalized response to Antonio for him to copy into IRS Solutions

**Pros:** Cheapest, fastest, no browser automation, works for any practice (not just IRS Solutions users) — broader distribution.

**Cons:** Disconnected from Command Room (no Pipeline integration, no Morning Brief surfacing). Lower workflow displacement. Not a defensible moat.

## Recommended Approach

**Phased: Approach A first (month 1), evolve to Approach B in month 2 alongside OLT prep work.**

### Rationale

- **Approach A ships month-1 value confidently** for Antonio's $1k/mo retainer expectation. He sees value (drafted responses prioritized in Morning Brief) within 30 days.
- **Validates the value prop before expensive integration.** If Antonio doesn't actually use the drafts, browser automation is a waste. Cheap test first.
- **Browser automation work in month 2 is concurrent with OLT prep** (which is also browser automation). Build the harness once, use for both wedges.
- **Approach C is interesting** for distribution-after-Antonio (works without IRS Solutions) but disconnected from Pipeline / Morning Brief — lower retention value, wrong v0 lead.
