## Section 2 — Drafting spec (how David transforms Antonio's answers into the case study)

After Antonio returns answers, David follows this drafting flow:

### Step 1 — Anonymization pass

Before writing, scrub every client-identifying detail. Use this checklist:

| Original field | Anonymization rule |
|---|---|
| Client name | "Client A" / "Client B" |
| Client business name | "[Industry] business" (e.g., "small construction business") |
| Client industry | Keep at sector level only (construction / restaurant / professional services / real estate / etc.) — never specific niche |
| Client geographic location | County or region only ("Riverside County" / "Bay Area") — never city + county combo |
| Client AGI | Bucket only (per the interview), never exact |
| Adjustment dollar amounts | Bucket only, never exact (use "approximately $X" framing) |
| Specific dates | Month + year only; never specific dates |
| IRS examiner name | Never include |
| Other professionals on the case (CPA, attorney) | Generic role only ("client's attorney" / "client's prior preparer") |

### Step 2 — Structure the narrative

Each case study follows a 6-section narrative arc:

1. **THE CLIENT** — 2-3 sentences. Anonymized client profile.
2. **THE NOTICE** — 2-3 sentences. What IRS sent, when, what's at stake.
3. **THE POSITION CHALLENGED** — 2-3 sentences. What the IRS is questioning + the dollar exposure.
4. **DOCKET IN THE WORKFLOW** — 1 paragraph. Which Docket capabilities Antonio used + the one most-useful moment.
5. **WITHOUT DOCKET, THIS WOULD HAVE BEEN** — 1 paragraph. What audit defense looks like by hand, vs with Docket's substrate.
6. **STATUS / OUTCOME** — 2-4 sentences. Current state + (if closed) outcome.

Plus a **callout box** with Antonio's quote.

### Step 3 — Voice rules (apply to every line)

- No em dashes
- No AI vocabulary (delve, crucial, robust, comprehensive, nuanced)
- Antonio's voice throughout — read aloud, replace any sentence that doesn't sound like him
- Real numbers, real dates (bucketized), real cites
- One claim per paragraph

### Step 4 — Review with Antonio

Before any external use:
1. David drafts the case study using Antonio's answers + this template
2. David sends the draft back to Antonio: "Here's the case study based on your answers. Read every sentence; flag anything inaccurate or any quote you don't want public. I'll iterate."
3. Antonio approves OR amends; iterate until both are happy
4. Capture the approved version with explicit written sign-off ("Antonio approves publication of this case study as of [date]. Approval scope: marketing materials, accelerator applications, pitch decks, landing pages, Discovery Scan PDFs.")
5. File the approval in `docs/legal/antonio-case-study-approval-{date}.md` (gitignored or restricted; the approval text in git is fine, attached signatures off-repo)

### Step 5 — Distribution

Once approved, the case study lands in:
- `docs/antonio-case-study-audit-1.md` (this file once published)
- `docs/antonio-case-study-audit-2.md`
- Cold outreach playbook update (per `docs/pitch-decks/cold-outreach-templates.md`)
- Pitch deck updates (per `docs/pitch-decks/*.md` — Slide N variations)
- Accelerator application updates (per `docs/accelerator-applications/*.md`)
- Landing page hero update (per `docs/landing-pages/homepage-copy.md`)
- Discovery Scan PDF footer (per `docs/discovery-scan-sample-output.md`)

### Word count target

Each published case study: **350-500 words including the quote**. Long enough to be substantive. Short enough that a YC reviewer reads the whole thing.

### Page footprint

Each case study fits on **a single page** (PDF rendering). At ~400 words: editorial-warm Fraunces 16-18pt body + Fraunces 24pt section headers + DM Sans 14pt quote → single-page layout with 0.75" margins.

### A/B variant tracking

After publishing, capture which case study (#1 vs #2) gets more engagement when used in:
- Cold outreach (open rate, reply rate, demo-booked rate)
- Pitch deck slide views (Loom analytics)
- Landing page hero (impression → form-submit conversion, once landing page ships)
- Accelerator application (impossible to track directly; surface via interview Q&A if accelerator asks about specific proof points)

Iterate next round of case studies (audit #3, #4 when they happen) toward the format that converts best.
